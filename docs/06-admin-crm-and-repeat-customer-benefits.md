# 관리자 CRM, 의뢰 목적 관리, 재의뢰 혜택 및 결제 연동 계획

## 1. 핵심 의견

관리자 화면은 단순히 "접수 목록을 보는 화면"이면 부족하다.

이 서비스는 녹취록 의뢰 특성상 고객의 상황, 의뢰 목적, 사건 배경, 긴급도, 결제 이력, 재의뢰 가능성을 함께 관리해야 한다.

따라서 관리자 화면은 `주문 관리`가 아니라 `사건 기반 고객 관리 CRM`으로 설계한다.

핵심 구조:

```text
고객
-> 사건
-> 사건 자료
-> 녹취 의뢰
-> 결제
-> 속기사 작업
-> 납품
-> 재의뢰/혜택
```

## 5. 회원제 운영 방향

이 서비스는 회원제를 운영하는 것이 좋다.

이유:

1. 사건 자료 보관이 필요하다.
2. 재의뢰 가능성이 높다.
3. 결제 이력과 할인 정책을 연결할 수 있다.
4. 법적/민감 자료 접근 이력을 관리해야 한다.
5. 고객별 상담 이력이 중요하다.

다만 처음부터 회원가입을 강하게 요구하면 이탈할 수 있다.

추천 흐름:

```text
파일 업로드
-> 의뢰 방식 선택
-> 견적 확인
-> 결제 전 휴대폰/이메일 인증
-> 자동 회원 생성 또는 간편 회원 전환
```

## 6. 2회차 의뢰 혜택

2회차 의뢰 혜택은 결제 시스템에 접목 가능하다.

대표적인 방식:

```text
첫 결제 완료 후 2회차 쿠폰 자동 발급
2회차 의뢰 시 결제창에서 자동 적용
관리자가 수동 할인 적용
AI 탐색비 무료 쿠폰 제공
일정 금액 이상 결제 시 정액 할인
```

### 추천 2회차 혜택

초기에는 복잡한 포인트보다 간단한 쿠폰형이 좋다.

```text
2회차 의뢰 5,000원 할인
또는
2회차 AI 구간 탐색 무료
```

이 서비스 특성상 `AI 구간 탐색 무료`가 더 자연스럽다.

왜냐하면 4,900원 탐색비는 고객에게 혜택으로 설명하기 쉽고, 실제 원가는 낮으며, 재의뢰를 유도하기 좋다.

추천 문구:

```text
첫 녹취록 의뢰 완료 고객님께 다음 의뢰 시 AI 구간 탐색 1회 무료 혜택을 드립니다.
```

## 7. 결제창에 혜택 적용 방식

결제창에서 자동으로 고객 혜택을 조회한다.

```text
고객 로그인/인증
-> 사용 가능한 혜택 조회
-> 결제 금액 계산
-> 혜택 자동 적용 또는 선택 적용
-> 최종 결제
```

결제창 예시:

```text
녹취록 작성 의뢰

기본금액: 66,000원
재의뢰 고객 혜택: -5,000원
최종 결제금액: 61,000원

[결제하기]
```

AI 탐색 무료 혜택 예시:

```text
AI 구간 탐색

기본금액: 4,900원
2회차 고객 혜택: -4,900원
최종 결제금액: 0원

[무료로 AI 탐색 시작]
```

## 8. 혜택 정책 주의사항

1. 할인 중복 여부를 정해야 한다.
2. 쿠폰 유효기간을 둬야 한다.
3. 환불 시 쿠폰 복구 여부를 정해야 한다.
4. 0원 결제는 PG 결제 없이 내부 처리해야 한다.
5. 관리자가 수동으로 혜택을 회수할 수 있어야 한다.
6. 남용 방지를 위해 휴대폰 번호 기준 중복 계정을 제한해야 한다.

## 9. 관리자 화면에서 고객 등급

초기 등급은 단순하게 둔다.

```text
신규
1회 결제
재의뢰
우수 고객
주의 고객
```

자동 기준:

```text
신규: 결제 0회
1회 결제: 결제 1회
재의뢰: 결제 2회 이상
우수 고객: 총 결제 30만원 이상 또는 3회 이상 의뢰
주의 고객: 관리자 수동 지정
```

## 10. 결론

고객 관리와 재의뢰 혜택은 결제 시스템에 충분히 접목 가능하다.

오히려 기존 200명 고객이 있다는 전제에서는 반드시 필요하다.

1차 MVP에서 복잡한 CRM을 모두 만들 필요는 없지만, 최소한 아래는 포함해야 한다.

```text
고객별 의뢰 횟수
고객별 결제 이력
의뢰 목적/의도 저장
관리자 상담 메모
2회차 혜택 발급
결제창 혜택 적용
```

이 구조를 넣으면 단순 접수 사이트가 아니라, 재의뢰와 고객관리가 가능한 녹취록 운영 플랫폼이 된다.

속기사 배정은 관리자 지정과 속기사 자율 선택을 병행할 수 있어야 한다. 다만 고객 만족도와 납품 품질을 위해 법원 제출용, 급행, 음질 불량, 다자간 녹취는 관리자 통제 또는 고등급 속기사에게만 노출하는 것이 좋다. 세부 정책은 `docs/07-transcriber-assignment-and-rush-fee-plan.md`를 따른다.
