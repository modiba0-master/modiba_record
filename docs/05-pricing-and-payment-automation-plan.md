# 요금표 기반 자동 견적 및 결제 유도 계획

## 1. 전제 변경

초기 검토 텍스트는 기존 200명 내외의 일상 고객이 확보되어 있다는 전제를 포함한다.

따라서 이 프로젝트는 완전한 무수요 상태에서 시작하는 신규 플랫폼이 아니라, 기존 고객을 디지털 접수/견적/결제 흐름으로 전환하는 프로젝트에 가깝다.

이 전제에서는 핵심 지표가 달라진다.

```text
무수요 신규 서비스:
방문자 확보, 광고 전환율, SEO가 핵심

기존 고객 보유 서비스:
기존 고객의 온라인 접수 전환율, 자동 견적 신뢰도, 결제 완료율이 핵심
```

즉, 1차 MVP의 목표는 신규 고객 대량 확보보다 `기존 고객의 접수/견적/결제 자동화`로 잡는 것이 현실적이다.

## 2. 이미지 요금표 기준 가격 구조

요금은 VAT 별도 기준으로 계산하고, 결제창에서는 VAT 포함 금액을 함께 보여준다.

| 녹음 시간 | 기본요금 | VAT 포함 결제금액 |
| --- | ---: | ---: |
| 3분 미만 | 20,000원 | 22,000원 |
| 5분 미만 | 30,000원 | 33,000원 |
| 10분 미만 | 60,000원 | 66,000원 |
| 20분 미만 | 80,000원 | 88,000원 |
| 30분 미만 | 100,000원 | 110,000원 |
| 40분 미만 | 130,000원 | 143,000원 |
| 50분 미만 | 160,000원 | 176,000원 |
| 60분 미만 | 180,000원 | 198,000원 |

60분 이상은 별도 가격 정책을 추가해야 한다.

## 3. 의뢰 방식별 계산식

### A. 전체 파일 녹취

```text
최종 요금 = 전체 작성 기본요금
```

예시:

```text
전체 파일 30분 미만
-> 100,000원
-> VAT 포함 110,000원
```

### B. 내가 아는 구간만 녹취

이미지 요금표 기준:

```text
구간 지정 작성 요금 = 전체 작성 기본요금의 20% + 선택 구간 요금의 120%
최소 결제금액 = 50,000원 VAT 별도
```

예시:

```text
전체 파일 30분
선택 구간 5분

(100,000원 x 20%) + (30,000원 x 120%)
= 20,000원 + 36,000원
= 56,000원 VAT 별도
```

### C. AI가 중요한 구간을 찾아주는 경우

이미지 요금표 기준:

```text
구간 탐색 후 작성 요금 = 전체 작성 기본요금의 50% + 선택 구간 요금의 120%
최소 결제금액 = 100,000원 VAT 별도
```

예시:

```text
전체 파일 60분
AI 탐색 후 10분 발견

(180,000원 x 50%) + (60,000원 x 120%)
= 90,000원 + 72,000원
= 162,000원 VAT 별도
```

### D. AI 구간 탐색비

```text
AI 구간 탐색비 = 4,900원 VAT 별도
```

정책:

```text
AI 탐색비는 최종 녹취록 의뢰 시 전액 할인 또는 차감한다.
```

이 문구가 중요하다. 사용자는 4,900원을 추가 비용이 아니라 최종 의뢰 전 선검토 비용으로 받아들인다.

## 4. 고객에게 보여줄 결제 유도 메시지

### 구간을 알고 있는 고객

```text
필요한 구간을 알고 계신가요?
직접 구간을 지정하면 전체 녹취보다 비용을 줄일 수 있습니다.
```

### 구간을 모르는 고객

```text
어느 부분이 중요한지 모르시나요?
AI가 먼저 주요 대화 구간을 찾아드립니다.
찾은 구간만 녹취 의뢰하면 전체 파일 녹취보다 비용을 줄일 수 있습니다.
```

### 4,900원 안내

```text
AI 구간 탐색 비용 4,900원은 최종 녹취록 의뢰 시 전액 차감됩니다.
녹취록을 의뢰하지 않아도 AI 탐색 결과는 확인할 수 있습니다.
```

## 5. 자동 견적 화면 흐름

### 1단계: 파일 업로드

시스템이 자동으로 확인한다.

```text
파일명
파일 형식
파일 크기
총 재생시간
```

### 2단계: 작업 방식 선택

```text
[전체 파일 녹취]
[내가 아는 구간만 녹취]
[어디가 중요한지 AI가 찾아주세요]
```

### 3단계: 자동 견적

전체 녹취:

```text
총 재생시간 28분 12초
적용 구간: 30분 미만
기본요금: 100,000원
VAT 포함 결제금액: 110,000원
```

직접 구간 지정:

```text
전체 파일: 30분 미만
선택 구간: 5분 미만

전체 기본요금 20%: 20,000원
선택 구간 요금 120%: 36,000원
최종 금액: 56,000원
VAT 포함: 61,600원
```

AI 구간 탐색:

```text
AI 구간 탐색비: 4,900원
최종 의뢰 시 전액 차감
```

AI 탐색 후 최종 견적:

```text
전체 파일: 60분 미만
AI 추천 구간: 10분 미만

전체 기본요금 50%: 90,000원
선택 구간 요금 120%: 72,000원
AI 탐색비 차감: -4,900원
최종 금액: 157,100원
VAT 별도 또는 포함 정책에 따라 표시
```

## 6. 여러 파일 견적 정책

여러 파일이 하나의 사건에 업로드될 수 있으므로 가격 정책을 명확히 해야 한다.

### 권장 1차 정책

MVP에서는 파일별로 기본요금을 계산하고, 사건 화면에서는 합산 견적을 보여준다.

예시:

```text
사건 A
파일 1: 10분 미만 -> 60,000원
파일 2: 5분 미만 -> 30,000원
파일 3: 20분 미만 -> 80,000원

전체 파일 녹취 합계: 170,000원 VAT 별도
```

이 방식이 좋은 이유:

1. 계산이 투명하다.
2. 고객이 파일별 비용을 이해하기 쉽다.
3. 긴 파일 하나와 짧은 파일 여러 개의 처리 난이도 차이를 반영할 수 있다.
4. 속기사 배정도 파일 단위로 나누기 쉽다.

### 2차 정책

장기적으로는 같은 사건의 여러 파일을 합산 시간 기준으로 할인할 수 있다.

```text
동일 사건 다중 파일 묶음 할인
반복 고객 할인
월 정기 고객 할인
```

다만 MVP에서는 자동 견적의 명확성이 더 중요하므로 파일별 계산 후 합산을 추천한다.

## 7. 기존 200명 고객 전제에서의 운영 전략

기존 고객이 있다면 1차 MVP는 광고용 랜딩페이지보다 접수 자동화가 더 중요하다.

우선순위:

```text
1. 기존 고객에게 보낼 의뢰 링크
2. 파일 업로드
3. 자동 재생시간 분석
4. 요금 자동 계산
5. 결제창 연결
6. 관리자 접수 확인
7. 속기사 배정
```

기존 고객에게는 이렇게 안내한다.

```text
이제 녹음파일을 보내주시면 재생시간 기준으로 자동 견적을 확인하고 바로 의뢰하실 수 있습니다.
필요한 구간을 모르실 경우 AI가 주요 구간을 찾아드릴 수 있습니다.
```

## 9. 주의할 점

1. VAT 별도/포함 표시를 혼동하면 고객 불만이 생긴다.
2. 4,900원 차감 정책을 명확히 해야 한다.
3. 최소 결제금액 50,000원/100,000원 적용 여부를 화면에 표시해야 한다.
4. 난이도 추가 비용은 자동 결제 전에 "사전 안내 후 진행"으로 둔다.
5. 60분 이상 파일은 별도 상담 또는 확장 요금표가 필요하다.
6. 여러 파일은 파일별 계산 후 합산이 MVP에 적합하다.
7. 회원별 재의뢰 혜택, 쿠폰, 관리자 수동 할인은 결제 계산 단계에 포함되어야 한다.

## 10. 회원 혜택과 결제 연동

결제 금액은 기본 요금표만으로 끝나지 않는다.

최종 결제 금액은 다음 순서로 계산한다.

```text
요금표 기준 금액
-> 최소 결제금액 적용
-> 난이도 추가요금
-> 급행 추가요금
-> AI 탐색비 차감
-> 쿠폰/재의뢰 혜택 적용
-> 관리자 수동 할인 적용
-> VAT 계산
-> 최종 결제금액
```

2회차 고객에게는 아래 혜택을 줄 수 있다.

```text
2회차 의뢰 5,000원 할인
또는
AI 구간 탐색 1회 무료
```

초기 추천은 `AI 구간 탐색 1회 무료`이다. 고객이 혜택을 체감하기 쉽고, 실제 원가 부담이 낮으며, 재의뢰를 자연스럽게 유도할 수 있다.

결제창에는 다음처럼 표시한다.

```text
기본금액: 66,000원
재의뢰 고객 혜택: -5,000원
VAT: 6,100원
최종 결제금액: 67,100원
```

고객 CRM과 혜택 정책은 `docs/06-admin-crm-and-repeat-customer-benefits.md`를 기준으로 한다.

## 11. 급행 의뢰 추가요금

요금표 이미지에는 `일반 3~5일, 긴급 작업도 가능`이라는 안내가 있으므로, 납기 옵션별 추가요금 정책이 필요하다.

초기 추천:

```text
일반 3~5영업일: 추가요금 없음
빠른 작업 2~3영업일: 20% 추가
긴급 작업 24시간 이내: 50% 추가
당일/야간/주말: 별도 상담 또는 80~100% 추가
```

24시간 이내와 당일 작업은 속기사 배정 가능 여부가 중요하므로, 자동 결제 전에 관리자 승인 또는 가능 속기사 확인 절차를 둘 수 있다.

속기사 배정과 급행 운영 정책은 `docs/07-transcriber-assignment-and-rush-fee-plan.md`를 기준으로 한다.

## 12. 결론

첨부 요금표는 자동 견적 시스템으로 구현하기 적합하다.

특히 기존 200명 고객이 있다는 전제에서는 `구간 탐색`보다 먼저 `업로드 후 자동 시간 분석 -> 즉시 견적 -> 결제` 기능이 매출 전환에 직접적이다.

따라서 1차 MVP 우선순위는 다음으로 조정한다.

```text
1. 파일 업로드
2. 재생시간 자동 분석
3. 요금표 기반 자동 견적
4. 결제창 연결
5. 관리자 접수 확인
6. AI 구간 탐색 옵션
7. 속기사 작업 화면
```
