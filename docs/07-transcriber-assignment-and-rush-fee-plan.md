# 속기사 배정 방식, 기량 관리, 급행 요금 정책

## 1. 핵심 의견

속기사 배정은 한 가지 방식만 쓰면 운영에 한계가 생긴다.

따라서 MVP부터 아래 두 가지 방식을 모두 지원할 수 있는 구조로 잡는 것이 좋다.

```text
1. 관리자 지정 배정
2. 속기사 자율 선택/수락
```

두 방식을 병행하면 일반 작업은 빠르게 배분하고, 어려운 작업은 관리자가 직접 통제할 수 있다.

## 2. 배정 방식 A: 관리자 지정 배정

관리자가 특정 속기사에게 작업을 지정한다.

적합한 경우:

```text
법원/기관 제출용
음질 불량
다자간 대화
전문용어 많음
급행 의뢰
고액 의뢰
민감 사건
신규 속기사에게 맡기기 어려운 사건
```

장점:

```text
품질 통제 가능
속기사 기량 차이 반영 가능
중요 사건 실수 방지
관리자가 책임 있는 배정 가능
```

## 3. 배정 방식 B: 속기사 자율 선택/수락

관리자가 작업을 공개하면 속기사가 목록에서 보고 직접 수락한다.

적합한 경우:

```text
일반 통화 녹취
짧은 구간 녹취
음질 양호
제출 목적이 낮은 일반 확인용
납기 여유 있음
```

장점:

```text
관리자 배정 업무 감소
속기사의 작업 선택권 보장
처리 속도 향상
여러 속기사 운영에 유리
```

단점:

```text
어려운 작업을 실력이 부족한 속기사가 가져갈 수 있음
인기 없는 작업이 남을 수 있음
긴급 작업이 제때 배정되지 않을 수 있음
```

따라서 자율 선택에는 제한 조건이 필요하다.

## 4. 자율 선택 제한 조건

속기사는 자신의 등급과 권한에 맞는 작업만 볼 수 있게 한다.

### 속기사 등급

```text
junior
standard
senior
legal_specialist
admin_verified
```

### 작업 난이도

```text
easy
normal
hard
legal
urgent
```

매칭 예시:

```text
easy/normal -> standard 이상
hard -> senior 이상
legal -> legal_specialist 이상
urgent -> admin_verified 이상 또는 관리자 지정
```

## 5. 작업 공개 범위

관리자는 작업 생성 시 배정 방식을 선택한다.

```text
[관리자가 직접 배정]
[조건에 맞는 속기사에게 공개]
[특정 그룹에게만 공개]
```

공개 작업에는 아래 정보만 보여준다.

```text
작업 유형
예상 분량
음질 상태
화자 수
납기
예상 작업료
제출 목적
난이도
```

고객 개인정보와 민감한 사건 설명은 수락 전 제한적으로만 보여준다.

## 6. 속기사 수락 흐름

```text
관리자가 작업 공개
-> 조건에 맞는 속기사에게 노출
-> 속기사가 작업 상세 확인
-> 수락
-> 작업 잠금
-> 제한 시간 내 시작
-> 최종 제출
-> 관리자 검수
```

작업 수락 후 일정 시간 안에 시작하지 않으면 자동 회수할 수 있어야 한다.

예시:

```text
수락 후 30분 내 작업 시작 없음
-> 관리자 알림
-> 자동 회수 또는 수동 회수
```

## 7. 관리자 개입 지점

자율 배정이라도 관리자가 언제든 개입할 수 있어야 한다.

```text
작업 회수
다른 속기사 재배정
속기사 권한 제한
납기 변경
추가 메모 전달
최종 승인 보류
```

특히 법원/기관 제출용은 속기사가 제출해도 관리자 최종 검수 후 납품하는 것이 좋다.

## 8. 급행 의뢰 요금

요금표 이미지에는 정확한 급행 산식이 수치로 명확히 들어가 있지는 않지만, `일반 3~5일, 긴급 작업도 가능`이라는 안내가 있다.

따라서 가격 정책에 급행 옵션을 추가해야 한다.

### 추천 납기 옵션

```text
일반: 3~5영업일
빠른 작업: 2~3영업일
긴급 작업: 24시간 이내
당일 작업: 당일 또는 별도 상담
```

### 추천 추가요금

초기 정책:

```text
일반: 추가요금 없음
2~3영업일: 기본 녹취 요금의 20% 추가
24시간 이내: 기본 녹취 요금의 50% 추가
당일/야간/주말: 별도 상담 또는 80~100% 추가
```

운영상 부담을 줄이려면 당일/야간/주말은 자동 결제보다 상담 후 확정으로 둔다.

## 9. 급행 요금 계산 순서

급행 요금은 할인 적용 전에 계산하는 것이 관리상 명확하다.

추천 순서:

```text
요금표 기준 금액
-> 구간 지정/AI 탐색 계산
-> 최소 결제금액 적용
-> 난이도 추가요금
-> 급행 추가요금
-> AI 탐색비 차감
-> 쿠폰/재의뢰 혜택
-> 관리자 수동 할인
-> VAT
-> 최종 결제금액
```

예시:

```text
기본 견적: 100,000원
24시간 이내 급행 50%: +50,000원
AI 탐색비 차감: -4,900원
최종 VAT 별도: 145,100원
VAT 포함: 159,610원
```

## 10. 급행 의뢰 운영상 주의점

급행 의뢰는 고객에게만 비싸게 받으면 되는 문제가 아니다. 속기사 배정과 품질 리스크가 함께 증가한다.

따라서 급행에는 아래 운영 정책이 필요하다.

```text
급행 가능 속기사만 배정
관리자 승인 후 접수
마감 시간 명확히 표시
속기사 작업료 가산
일정 시간 내 미수락 시 관리자 알림
납품 전 관리자 최종 검수
```

## 13. 추가 의견

속기사의 기량 차이는 실제 운영에서 반드시 문제가 된다. 그러므로 완전 자율 배정만으로 가면 품질 사고가 날 수 있다.

추천 운영 방식:

```text
일반 작업: 자율 선택 가능
법원/기관 제출용: 관리자 지정 또는 senior 이상만
급행 작업: 관리자 승인 후 배정
음질 불량/다자간: senior 이상만
신규 속기사: easy 작업만
```

이 구조가 가장 현실적이다.

## 14. 결론

관리자 지정 배정과 속기사 자율 선택은 병행하는 것이 좋다.

다만 자율 선택에는 등급, 전문분야, 급행 가능 여부, 법원 제출 가능 여부 같은 제한을 반드시 둬야 한다.

급행 요금은 가격 정책에 별도 반영해야 하며, 24시간 이내 의뢰는 단순 자동결제보다 관리자 승인과 속기사 가능 여부 확인이 필요하다.
